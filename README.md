> **Upload a resume → get a production-ready portfolio in seconds. 100% browser-based, zero backend.**

[![CI](https://img.shields.io/github/actions/workflow/status/shaikhshahnawaz13/portfolioforge/ci.yml?style=for-the-badge&label=CI%2FCD&logo=github-actions&logoColor=white)](https://github.com/shaikhshahnawaz13/portfolioforge/actions)
[![Stars](https://img.shields.io/github/stars/shaikhshahnawaz13/portfolioforge?style=for-the-badge&color=FFD700&logo=github)](https://github.com/shaikhshahnawaz13/portfolioforge/stargazers)

<table>
<tr>
<td width="50%">

**🔑 Key Features**
- 🤖 **4 AI Providers** — Groq, Gemini, OpenAI, OpenRouter
- 🔄 **Key Pool Rotation** — auto-rotates on rate-limit
- 📄 **In-Browser Parsing** — PDF.js + Mammoth.js, zero uploads
- 🎨 **6 Design Presets** — Dark, Editorial, Terminal, Glass, Bold, Clean
- 📦 **One-Click Export** — downloads ZIP with `html/css/js`
- 🔒 **Privacy-First** — keys in JS memory only, never stored

</td>
<td width="50%">

**🧪 Engineering Quality**
- ✅ **36 Unit Tests** — parsers, validators, error classes
- 🚀 **GitHub Actions CI/CD** — auto-deploys to Pages
- ♿ **Fully Accessible** — ARIA roles, keyboard nav, `aria-live`
- ⚡ **Zero Build Step** — vanilla HTML/CSS/JS, instant deploy
- 🎯 **4-Strategy Parser** — handles LLM output drift gracefully
- 🔁 **Retry Architecture** — round-robin backoff over key pool

</td>
</tr>

<details>
<summary><b>🏗️ Architecture Deep-Dive (click to expand)</b></summary>

<br/>

```
portfolioforge/
│
├── index.html                # App shell: semantic HTML + ARIA, zero inline styles
│
├── src/
│   ├── style.css             # Design system via CSS custom properties
│   │   ├── :root {}          # 20+ variables: colors, fonts, radius, transitions
│   │   ├── animations        # sweep, blink, reveal-up, fade-in, badge-in
│   │   └── responsive        # 768px + 480px breakpoints, mobile-first
│   │
│   └── app.js                # All logic — vanilla ES2022, zero runtime deps
│       ├── STATE {}           # Single source of truth
│       ├── PROVIDERS {}       # Per-provider config + validation
│       ├── Key Pool Engine    # addKeyToPool → markRateLimited → getNextKey
│       ├── callWithKeyRotation()  # Round-robin + multi-round backoff
│       ├── extractPDF()       # PDF.js page-by-page text reconstruction
│       ├── extractDOCX()      # Mammoth.js raw text extraction
│       ├── cleanResume()      # Normalise + hard-cap at 1,800 chars
│       ├── callGemini()       # Gemini-specific request + error mapping
│       ├── parseAI()          # 4-strategy: delimiters → fences → normalised → raw
│       └── renderPreview()    # Sandboxed srcdoc iframe with injected CSS + JS
│
├── tests/app.test.js         # 36 tests across 6 describe blocks
└── .github/workflows/ci.yml  # test → validate → deploy pipeline
```

</details>

## 📐 Engineering Philosophy

<div align="center">

```
"Ship it, then improve it.
 Tests first on anything that has state.
 If it needs a server, question why.
 If it needs a framework, question harder."
```

</div>

<br/>

<div align="center">

| Principle | Practice |
|:---|:---|
| 🎯 **Simplicity over cleverness** | Readable 10-line function beats a smart 3-liner |
| 🚢 **Ship early, iterate fast** | Working v1 beats perfect v0 every time |
| 🧪 **Test behaviour, not implementation** | Tests describe *what*, not *how* |
| 🔒 **Security by default** | Keys in memory only · HTTPS always · no implicit trust |
| 📦 **Zero-dependency where possible** | Vanilla JS over a framework for focused tools |
| 📝 **Docs are code** | README, inline comments, and types are first-class citizens |

</div>
